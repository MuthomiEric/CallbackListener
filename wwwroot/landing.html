The problem it solves
// Your webhook sender (Stripe, GitHub, Twilio...) needs a // public URL. localhost:3000 isn't reachable from the internet. POST https://callback.erickmuthomi.dev/webhooks/stripe?slug=payments ↓ received by the relay server ↓ agent running on your machine picks it up POST http://localhost:3000/webhooks/stripe ↑ arrives as if Stripe called you directly
Features
realtime_feed()
Every incoming webhook appears instantly. Headers, body, query params, source IP — all visible.
relay_to_local(port, path)
Install the agent once. It stays connected and forwards callbacks to whatever port and path you configure.
delivery_mode: "Both"
Web only, local only, or both. Switch per app, any time. Local mode keeps callbacks off the web feed entirely.
rate_limit(60, "per_slug")
60 requests per minute per slug. Global IP-based limit on top. No runaway webhook floods.
Quick test (no account needed)
Send a webhook to the public demo endpoint and inspect it instantly after signing in.
# Send a test callback to your app slug curl -X POST "https://callback.erickmuthomi.dev/test?slug=your-slug" \ -H "Content-Type: application/json" \ -d '{"hello":"world","source":"curl"}'
